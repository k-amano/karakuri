# 改善事項バックログ

単体テスト初回正常終了後に確認された改善事項の一覧。優先順位付き。

**最終更新**: 2026-04-14

## 優先度: 高（動作品質・致命的な問題）

### H1: 文字化け ❌ 未対応

- **現象**: 日本語テキストの一部が文字化けする
- **例**: 「英���→日本語」「入��UI」
- **原因**: ストリーミング受信時のUTF-8マルチバイト文字がチャンクの境界で分割されることが原因と推定
- **対応方針**: フロントエンドの `TextDecoder` を `stream: true` オプション付きで正しく使用しているか確認・修正

### H2: テスト結果が画面に表示されない ❌ 未対応

- **現象**: テスト完了メッセージだけが出て、パス数・失敗数などの集計が画面に見えない
- **補足**: ログには流れているが視覚的に強調されていない
- **対応方針**: 実装確認パネル（`reviewing` 状態）にテスト結果サマリー（passed / failed 件数）を表示する

### H3: テストケース修正UIの不備 ❌ 未対応

- **現象**: 「修正を依頼」ボタンはあるが、修正内容の入力欄が `window.prompt`（ブラウザ標準ダイアログ）になっており使いにくい
- **要件**: 修正希望内容をメッセージとして送る形式（テキストエリア + 送信ボタン）に変更する
- **対応方針**: テストケース確認パネルにインライン入力欄を追加し、`window.prompt` を廃止する

## 優先度: 中（機能上の問題）

### M1: テスト種別が画面上で不明 ❌ 未対応

- **現象**: ステータスバッジやバナーに「テスト」とだけ表示され、単体テスト / 結合テスト / E2Eテストのどれかが分からない
- **対応方針**: `running_tests` の状態に `test_type`（unit / integration / e2e）を持たせ、バナー・バッジに反映する

### M2: 処理中の進捗表示がない ❌ 未対応

- **現象**: 処理中に動いているか止まっているか分からない
- **要件**: 処理内容（テストコード生成中 / テスト実行中 / 自動修正中 1/3 など）またはステップをバナーに表示する
- **対応方針**: 各フェーズの開始時にバナーテキストを更新する

### M3: テストケースの出力形式 ❌ 未対応

- **現象**: 現在はMarkdownのチェックリスト形式で出力される
- **要件**: 以下の列を持つ表形式に変更する
  - 対象画面
  - テスト項目
  - 操作方法
  - 期待動作
- **対応方針**: `generate_test_cases()` のプロンプトを修正し、Markdown表形式で出力させる

### M4: テスト結果の集計表 ❌ 未対応

- **現象**: ログにテキストで流れるだけで、集計表がない
- **要件**: テストケース一覧に結果・判定・実行日時の列を追加した集計表を作成する
- **対応方針**: 実装確認パネルおよびテストレポートのMarkdownファイルに集計表を追加する

---

## 優先度: 低（UX改善）

### L1: 右ペインのUIレイアウト刷新 ❌ 未対応

- **現象**: メッセージ入力欄が狭い。処理フェーズのタイミングによって入力欄が消えることがある
- **要件**: GitHubのIssueのように、会話履歴が上から時系列で積まれてスクロールし、入力欄は常に下部に固定されるチャット型レイアウトに変更する
- **対応方針**: `instruction-panel` のレイアウトを全面的にチャット型に刷新する

### L2: 前回の続きから再開 ✅ 対応済み（2026-04-14）

- **内容**: 実装 → 単体テスト → 結合テスト → E2Eテストのステップで、完了済みのステップをスキップして続きから再開できる
- **実装内容**:
  - `GET /instructions/last-completed` エンドポイントを追加
  - タスク詳細画面ロード時にテスト実行履歴を取得し、完了済みステップを `PromptState` に復元
  - 再開バナー（「最初からやり直す」ボタン付き）を表示
